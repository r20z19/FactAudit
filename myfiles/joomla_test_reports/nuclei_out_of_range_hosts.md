### Joomla 1.5.x 系列 (6 主机) — 完全不受影响

**结论:** 所有 13 个漏洞的最低受影响版本为 Joomla 2.5（漏洞五）或更高。Joomla 1.5.x（发布于 2008-2012 年）**全部不适用**。

- 这些版本没有 REST API 功能（Joomla 4.0+ 才引入）
- 缺少 com_finder 智能搜索组件（从 2.5 引入）
- 缺少 com_fields 自定义字段组件（从 4.0 引入）
- 缺少 com_joomlaupdate 组件（从 2.5 引入，但漏洞七需 4.0.4+）
- 底层框架差异大（MooTools 而非 jQuery、Bootstrap 未集成）

### Joomla 2.5.x 系列 (12 主机) — 仅漏洞五理论上适用

**结论:** 仅**漏洞五 (com_finder 反射型 XSS)** 理论上版本范围覆盖（从 2.5 起），其余 12 个漏洞均不适用。

适用性分析:
- **漏洞五 (com_finder):** 理论适用。com_finder 组件在 2.5 中引入，前端默认对访客开放。搜索词回显逻辑的未转义输出问题在 2.5 系列中同样存在。但实际利用受 finder 默认分词器对 HTML 特殊字符的剥离限制。
- 漏洞三/四/六/八：需要 3.x+，不适用
- 漏洞一/二/七/九~十三：需要 4.0+，不适用

### Joomla 3.x 系列 (108 主机) — 5 个漏洞理论上适用

**结论:** 3.x 系列**缺少 Joomla 4.0+ 引入的 REST API**，因此漏洞二/九至十三（全部为 API 越权类）不适用。漏洞一（com_fields）和漏洞七（Zip Slip）也需要 4.0+，同样不适用。

**理论适用的 5 个漏洞:**

| 漏洞 | 名称 | 所需权限 | 实际利用可行性 |
|:----:|------|---------|:------------:|
| 漏洞三 | com_templates deleteFile | Super Admin / Admin | 需管理员后台访问，注册通常关闭 |
| 漏洞四 | com_banners custombannercode XSS | Editor+ | 需 Editor 权限，注册通常关闭 |
| 漏洞五 | com_finder 反射型 XSS | **无**（访客可访问） | 部分可行，但分词器剥离 HTML 字符 |
| 漏洞六 | PageBreak 存储型 XSS | Editor+ | 需 Editor 权限，注册通常关闭 |
| 漏洞八 | com_templates copyFile | Admin+ | 需管理员后台访问，注册通常关闭 |

**关键限制:**
1. **认证壁垒:** 漏洞三/四/六/八 均需要 Editor 或 Admin 级别认证。3.x 系列的用户注册配置通常全部关闭。

2. **漏洞五无认证要求但利用受限:** com_finder 在前端默认对访客开放。3.x 系列同样使用 findermin 分词器，其默认配置剥离 HTML 特殊字符 (`<` `>` `"` `'` `=` `(` `)` `{` `}` `&`)。

3. **无 REST API:** 3.x 不具备 Joomla 4.0+ 的 REST API (`/api/index.php/v1/*`)，API 相关漏洞全部不适用。

4. **已知的 3.x 特有 CVE 未在本报告中涉及:** 本报告分析基于 13 个待分配 CVE 的漏洞报告。3.x 系列有其他已知漏洞（如 CVE-2023-23752 信息泄露、CVE-2022-23750 等）不在此 13 个漏洞范围内。

**此前已测试的 3.x 实例的印证:** dev.reactome.org (Joomla 3.10.12) 的测试（见 dev.reactome.org.md 报告）验证了上述分析：3.x 系列无 REST API 端点，register 关闭，需认证的漏洞均无法利用。com_finder 存在但自定义搜索入口。

### 综合结论

这 126 个范围外主机**均不适合作为 13 个待分配 CVE 的 CNVD 申请验证资产**。主要原因:

1. **版本差距过大:** 85.7% 为 3.x、9.5% 为 2.5.x、4.8% 为 1.5.x，均低于最低受影响版本 4.0.0
2. **API 漏洞完全不可用:** 漏洞二/九至十三（共 6 个需 REST API 的漏洞）在全部 126 个主机上均不适用
3. **需要 4.0+ 的漏洞不可用:** 漏洞一 (com_fields) 和漏洞七 (Zip Slip) 同样不适用
4. **认证壁垒:**
   - 理论适用于 3.x 的 5 个漏洞（漏洞三/四/五/六/八）中 4 个需 Editor+/Admin+ 权限
   - 3.x 站点用户注册同样默认关闭，与已测试的 4.0+ 站点情况一致
5. **唯一无认证要求的漏洞五利用受限:** finder 分词器限制在所有 Joomla 版本中一致
6. **主机不可达风险高:** 这批主机多为 IP 地址（125/126 为 IP，仅 112-magazin.de 和 10th.hydrogeologycongress.gr 为域名），其中部分可能已经下线或迁移
