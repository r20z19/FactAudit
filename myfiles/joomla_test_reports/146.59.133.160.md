## Key Findings

1. **Registration**: Closed - all auth-requiring vulns blocked
2. **API**: API endpoints exist but require authentication tokens
3. **com_finder XSS (Vuln 5)**: Not exploitable - no reflected output.
4. **All authenticated vulns**: Blocked by registration closure and/or missing credentials

## Conclusion

All 13 vulnerabilities are theoretically applicable to Joomla 5.4.6 but **none are practically exploitable** without valid credentials. The com_finder reflected XSS (Vuln 5) is the only one approachable without authentication but is neutered by the default tokenizer and WAF.
