## Key Findings

1. **Registration**: HTTP 303 - registration disabled
2. **API**: API endpoints exist (/v1/users, /v1/languages) but require authentication tokens (401). Endpoints are valid.
3. **com_finder XSS**: Accessible (HTTP 200) but tokenizer strips all HTML special characters (< > " = ( ) { } &) — code-level vulnerability confirmed but no practical XSS exploitation possible
4. **Authenticated vulns**: ❌ Blocked (no registration)

## Conclusion

All 13 vulnerabilities are theoretically applicable to Joomla 4.x but **none are practically exploitable** without valid credentials. com_finder is accessible (differing from many other API_4.x hosts) but tokenizer prevents exploitation. /administrator/ blocked by 403.
