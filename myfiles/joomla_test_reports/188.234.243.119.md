## Additional Intel

- Homepage: 200 OK (28347 bytes)
- Registration page returned HTTP 200 with `id="member-registration"` form elements, but registration requires email activation — no low-privilege account was created
- /administrator/ returns 403 Forbidden (blocked)
- API v1 endpoints (/users, /languages) return 401 — endpoints exist but require authentication tokens
- com_finder returns 404 (not available on this instance)

## Key Findings

1. **Registration**: Registration page is reachable (HTTP 200) with valid form, but no working low-privilege account was obtained
2. **API**: API endpoints exist (/v1/users, /v1/languages) but require authentication tokens (401)
3. **com_finder XSS**: No com_finder endpoint available (404) — cannot test
4. **Authenticated vulns**: ❌ Blocked (no credentials available)

## Conclusion

All 13 vulnerabilities are theoretically applicable to Joomla 4.x but **none are practically exploitable** without valid credentials. API endpoints confirmed functional (returning 401), confirming Joomla 4.x REST API is present. com_finder not available. /administrator/ blocked by 403.
