## Key Findings

1. **Registration**: ✅ Registration form confirmed with 'Register' button and email field at HTTP 200
2. **API**: API endpoints exist (/v1/users, /v1/languages) but require authentication tokens (401). Endpoints are valid.
3. **com_finder XSS**: query-explained + span.term detected - code-level vulnerability confirmed. However, the default findermin tokenizer strips HTML special characters (< > \" ' = ( ) { } &), preventing practical XSS exploitation.
query-explained block visible but search terms not reflected in span.term.
4. **Homepage components**: com_fields=not detected, com_banners=not detected, PageBreak=not detected
5. **Authenticated vulns**: Potentially testable via registration

## Conclusion

Registration is OPEN on this host. This is significant because it means an attacker could register a user account and potentially exploit authenticated vulnerabilities (Vulns 1, 3, 4, 6, 7, 8). Further manual testing with a registered account is recommended.
